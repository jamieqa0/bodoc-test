## Bodoc QA Dashboard 개선점

분석 결과 UI/UX, 기능, 성능, 접근성 등 다양한 영역에서 개선할 부분을 발견했습니다.

---

### 1. 반응형 디자인 — 심각

현재 CSS 미디어 쿼리가 **0개**입니다. 사이드바가 240px 고정이고 main 영역이 flex로 되어 있지만, 화면 크기에 따른 대응이 전혀 없습니다. 테스트 결과 페이지에서 리포트 패널의 텍스트가 세로로 깨져서 읽을 수 없는 수준이었고, 테스트 실행 페이지에서는 실시간 로그 패널과 시나리오 상태(Idle) 표시가 잘려 나갑니다. 최소한 1024px, 768px 등 주요 브레이크포인트에서 레이아웃이 정상 동작하도록 미디어 쿼리를 추가해야 합니다. 사이드바도 좁은 화면에서는 접을 수 있게(collapse) 만들면 좋겠습니다.

---

### 2. 접근성(Accessibility) — 심각

aria-label, aria-role이 모두 **0개**이고, 41개 이미지 중 alt 속성이 있는 것이 **하나도 없습니다**. tabindex도 없어 키보드 내비게이션이 불가능합니다. label의 for 속성 연결도 없고, 입력 필드 중 label이 없는 것도 있습니다. 스크린 리더 사용자를 고려하지 않더라도, 기본적인 ARIA 속성과 alt 텍스트(특히 테스트 스크린샷에 시나리오명 + 스텝 설명을 넣으면 유용함)를 추가하는 것이 좋습니다.

---

### 3. 네트워크 폴링 → WebSocket/SSE 전환

현재 로그, Appium 상태, 디바이스 상태, 테스트 상태를 **매 1초마다 4개의 HTTP GET 요청**으로 폴링하고 있습니다. 테스트가 15분 정도 돌아간다면 약 3,600개의 불필요한 요청이 발생합니다. 이미 브라우저에 WebSocket과 EventSource가 지원되므로, 실시간 로그와 상태 업데이트는 WebSocket이나 SSE(Server-Sent Events)로 전환하면 서버 부하와 네트워크 트래픽을 크게 줄일 수 있습니다.

---

### 4. 삭제/비우기 작업에 확인 다이얼로그 없음

"삭제" 버튼(이전 결과 삭제)과 "비우기" 버튼(로그 비우기)에 confirm 다이얼로그가 없습니다. 실수로 클릭하면 테스트 결과가 영구 삭제됩니다. 특히 이전 결과 삭제는 16건 이상의 데이터가 한번에 사라질 수 있으므로, 최소한 "정말 삭제하시겠습니까? (N건)" 같은 확인 절차가 필요합니다.

---

### 5. 대시보드 지표 개선

현재 4개 지표(전체 실행 횟수, 성공률, 최근 실패 건수, 등록 시나리오)가 있지만 더 유용한 정보를 추가할 수 있습니다. 성공률 추이 차트(시간 경과에 따른 트렌드), 시나리오별 실패 빈도(어떤 시나리오가 가장 불안정한지), 평균 실행 시간, 마지막 성공 실행 시각 등을 추가하면 QA 현황을 한눈에 파악하기 훨씬 좋아집니다.

---

### 6. 검색 및 필터링 기능 부재

실행 이력 테이블에 검색이나 날짜 범위 필터가 없습니다. 테스트 결과가 쌓이면 특정 날짜나 특정 시나리오 실패만 찾기 어려워집니다. 날짜 범위 필터, 시나리오 이름 검색, PASS/FAIL 필터(테스트 결과 페이지에는 있지만 대시보드에는 없음)를 추가하면 좋겠습니다.

---

### 7. 페이지네이션 없음

실행 이력 테이블에 페이지네이션이 없습니다. 현재는 10건 정도라 괜찮지만, 장기 운영 시 수백 건의 이력이 쌓이면 페이지 로딩이 느려질 수 있습니다. 페이지네이션 또는 무한 스크롤을 도입하고, 한 페이지당 표시 건수를 설정할 수 있게 하는 것이 좋습니다.

---

### 8. 리포트 내보내기(Export) 기능 없음

테스트 결과를 PDF, HTML, CSV 등으로 내보내는 기능이 없습니다. QA 리포트를 다른 팀원이나 관리자에게 공유하려면 현재는 스크린샷을 찍는 수밖에 없습니다. 실행 리포트를 HTML 또는 PDF로 다운로드하는 기능, 시나리오별 성공/실패 데이터를 CSV로 내보내는 기능을 추가하면 실용성이 크게 올라갑니다.

---

### 9. Favicon 없음

브라우저 탭에 기본 아이콘이 표시됩니다. 간단한 개선이지만, 여러 탭을 열어놓고 작업할 때 식별성이 떨어집니다. 보닥 로고나 QA 관련 아이콘을 favicon으로 설정하면 좋겠습니다.

---

### 10. 실패 시나리오 재실행(Retry) 기능 없음

실패한 시나리오만 선택적으로 재실행하는 기능이 없습니다. 현재는 전체 실행 또는 개별 Run 버튼만 있는데, 테스트 결과 페이지에서 실패한 시나리오만 모아서 "실패 재실행" 버튼을 제공하면, 10개 중 2개만 실패했을 때 15분 전체를 다시 기다릴 필요가 없어집니다.

---

### 11. 시나리오 실행 순서 변경(Drag & Drop) 불가

시나리오 목록의 순서를 변경할 수 없습니다. 특정 시나리오만 먼저 테스트하고 싶을 때 드래그 앤 드롭으로 순서를 바꾸거나, 체크박스로 복수 선택 후 선택 항목만 실행하는 기능이 있으면 편리합니다.

---

### 12. 에러 메시지 가독성

실패 원인의 스택트레이스가 한 줄로 이어져 있어 읽기 어렵습니다. 스택트레이스를 코드 블록 형식으로 줄바꿈해서 표시하고, 핵심 에러 메시지를 상단에 강조 표시하면 디버깅 시간을 줄일 수 있습니다.

---

### 13. 로딩/진행 상태 표시 부족

시나리오 실행 중일 때 각 시나리오 카드에 진행 상태(Running, Passed, Failed)를 색상이나 애니메이션으로 실시간 표시해주면 좋겠습니다. 현재는 "○ Idle"만 표시되고, 상단 배지에 "시나리오 실행 중 (4/10)"으로만 나옵니다. 현재 실행 중인 시나리오에 스피너나 프로그레스 바를 추가하면 진행 상황을 직관적으로 파악할 수 있습니다.

---

### 14. 다크 모드 미지원

시스템 다크 모드 미디어 쿼리(`prefers-color-scheme`)를 감지하지 않고 있습니다. 로그 패널은 이미 어두운 배경인데 나머지는 밝은 테마라 일관성이 없습니다. 다크/라이트 모드 토글을 설정 페이지에 추가하면 야간 작업 시 눈의 피로를 줄일 수 있습니다.

---

### 15. 멀티 디바이스 지원 UI 부족

현재 1대 연결로 표시되고 있는데, 추후 여러 디바이스에서 병렬 테스트를 하게 될 경우 디바이스별 테스트 결과 분리, 디바이스 선택 후 실행 등의 기능이 필요합니다. 설정 페이지에서 디바이스를 선택하는 드롭다운 정도만 미리 준비해 두면 확장성이 좋아집니다.
